Errata

    There are a few html elements with tags here you should recognize.
    We're using the TABLE element for this example. Generally, tables should be used for tabular data, like a spreadsheet.
    Historically, they've been used for layouts and even as a way to compress an image in different blocks.
    There are new and complicated ways to lay things out regularly in HTML
    like flexbox and CSS grid, but for starting out
    this is going to be much more straightforward. 
    
    #hightlight TABLE and ROW
    We have a TABLE element here.
    Tables have rows, which are denoted by the TR tag, T.R. table row.
    Each row in the table is a TR element.
    For now we have one.
    Within each row, we have TDs, table data. In a regular grid, you have the same number of TDs in every row,
    but tables can get complicated, with COLSPANS and ROWSPANS to join TDs together.
    For this assignment, it's important to know that TRs and TDs will grow to fit their contents,
    They also obey CSS rules.
    

    In the TDs we have IMG tags, IMG for image.
    Notice that these don't have closing tags. They don't need them. Most tags do!
    The src attribute contains a path to the image, either a RELATIVE path, like these. Relative paths give directions relative
    from the path of the file the src tag is in.  In this case, it's in a folder called img.  If you download this code,
    you can put your own images in that folder, and point to them similarly.

    I screenshotted and clipped out just two of Muybridge's images using the technique Professor Sack 
    described in the introduction video. They're in the same folder as this talkthrough so I can access them using a 
    relative path in the SRC attribute.

    Now, the third and last row is a bit different.  What's going on here?
    The first TD in the row has an attribute, COLSPAN, set to 2, which makes it double-wide. It spans two columns.
    
    Because the image inside is twice as wide, in order to maintain its correct aspect ratio, it's also taller. 
    This makes the remaining cell square (or nearly square). 
    In this row I've added CSS classes to the image in the first cell using the class attribute. 
    You can call a class anything as long as it matches with the CSS matches later. An IMG tag is always an IMG tag,
    but you can call a class "wide" or "sblaglorgabob", short and descriptive is usually best.
    
    There's also a class attached to the last TD. Notice that it doesn't have an image tag in it. 
    Where is it coming from?  To answer that, let's look at the CSS.

    #switch tab
    css is a set of selectors, which point to the part of the HTML you want to style, and then a set of rules between
    braces. 

    Starting at the beginning, we have a selector for the body of our HTML, which is the full content area.
    I've set the background to black. The CSS interpreter knows that black is a color. HTML has a lot of reserved keywords
    that you can use to select a color.  Black is black.  Blue is blue. papayaWhip is the color of papaya whipped in cream. 

    I've also added borders to the table and TD elements, 
    this can be useful when laying things out so you can see their boundaries more clearly.  For this border I've useda 
    different coding of color. Rather than a keyword, I've set the values of red green and blue using a rgb color.
    These are numbers between 0 and 255.  0 0 0 is black, 255 255 255 is white.

    I've used a similar but different color coding for TD coloration, which uses pairs of hexidecimal numbers, where you count
    using numbers and letters, 0-9 a-f. f is 16. #9900cc is the same as rgb(60,0,80). There are other codings as well:
    cmyk, rgba -- which is like rgb but with transparency, hsb for hue saturation brightness. Take a look at the MDN
    documentation for more info.

    I've set the width of the image to 30 vw, vw is a fractional unit of the width of the view window, 30 is just under 1/3.
    If I use the image at its native resolution, it's a little too wide for the viewer on my laptop.
    If the HTML wasn't sharing the screen with the editor, it would fit without this rule. See what works best for you.
    
    The image with class "wide" has its width set to 100%, that's 100% the size of its parent element, the TD.
    Without that it would match the default height of the table cell. Because I'm not setting a height
    it scales based on its aspect ratio, if you do set a height, you can create some interesting distortions. Try it!
    Like color, width takes lots of different kinds of units. Percentages. pixels (px), ems and rems.
    
    The last cell, with the "square" class has its background set using CSS. This is a way to put - or fill an element
    with an image without using the IMG tag. If there were text or other content in this element, 
    the background-image would, as the property suggests, sit behind it. 

    The background-image property usually takes a url that points to an image file using the format I've used here, 
    but it can also take a gradient as an input. 
    
    The background-size property takes inputs of width and height, I'm setting it to
    25% height, which is calculated from the element it's in, and letting it figure out the width.
     
    by default the image repeats within the element it's applied to, 25% is a quarter, so we get it 4 times.
    there are ways to limit or prevent repetition, but I like the effect. Very Andy Warhol.
