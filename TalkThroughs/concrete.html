on blank space

haha line break machine go br

I use a couple <br> tags in this project, but I didn't discuss them in the talkthrough

BR stands for break, and inserts a single line break

In the early days of the web, BR tags were used to add blank space to the web. We have many more tools to do this now than we did back then (most of which are a bit more elegant), and they've fallen out of fashion

But for creating vertical space in a poem? They are a good way to do that. You could also add a class that adds margin-top or padding-top to an element.

the <pre> tag is for preformatted text, and any blank space inside it will be preserved

To add blank space within                 a line of text (like I used here), you can also type out &nbsp; which is a special character called "non-breaking space". You also need special characters for &, > and other otherwise-reserved characters in html. Here's a list of them from the w3c.

note: I am using the term 'blank space' over what was customarily referred to as 'white space' in graphic design, where white refers to the most ubiquitous background color of paper. Colors in HTML are far more variable.

      Alright, let's talkthrough the code for this concrete poem, or this non-concrete poem that I have concretized.
       
      By this time, you should already be familiar with the structure of HTML and some of the basics of using CSS to position
      elements. If you haven't yet, check out the resources assigned for this week.
      
      Before I start moving through the code, I'm going to talk broadly about the structure and briefly about the structural elements that make up this poem, and a little about the block and inline display modes.
      
      the first tags used in this page, for the poem's title and attribution, are heading tags, which in html are numbered h1 through h6, where h1 is for top level headings and the lower numbers for subheadings.
      
       each line of the poem is surrounded by block element tags, either div or p (p stands for paragraph).  block elements start on a their own new line, and by default p tags have a line's worth of margin above them. I don't use a p for every line because a some of the lines contain divs, and you're not supposed to put additional block elements inside a paragraph.
      
      i also use div tags to group parts of the poem that i want to style together; divs are a generic block level element.
      by generic I mean 'div' doesn't really signify a purpose in the way that paragraph or image does. you can think of them
      structurally as a block of html that can contain anything, including other divs and tags, or nothing. An empty div tag can still have height and width
      and, a background, and other properties applied using CSS.
      
      similarly, i am using spans to group parts of lines of text for styling. spans are also generic, but unlike divs they
      are inline elements.  inline elements do not break the line, so they are good for styling within a cluster of text or other inline elements.
      
      if this feels a bit confusing, try changing some of my spans into divs and see what happens (but don't forget to change the closing tags, too)
      
      i haven't applied any stying to the p tag itself, instead I'm using classes to do this work.
      
      we'll look at the CSS a bit later, but before we do, I will point out that the only tag I have styled directly is
      the body to change the background color to e5e5e5, which is a little less white than it would be by default.
      
      all the other styling that's happening on the page here is accomplished using classes. 
      
      rather than talking through the HTML from top to bottom and then the CSS from top to bottom like I did with the tables,
      I'm going to be moving back and forth a bit.
      
      Finally, before I start - a bit about the poem that I've excerpted here. Diane di Prima's Revolutionary Letters 
      was published in 1971
      by City Lights Books. I chose this poem because it speaks, I think, quite directly to this moment. It also asks the reader
      to think about space and position, which is also what this assignment is about at a much more formal level.

      Our HTML starts out as it often does, doctype, language, which should be set for accessibility reasons.

      in our head we have our title,
      and links to 2 CSS files, the first one we'll find in the CSS
      the second is an embed from Google Fonts. We'll talk about that a little later.

  next, in the body: 
      the title and author are wrapped in heading tags, h1 and h2; in headers h1 is the largest
      default size, h6 is the smallest.

      both h3 and h4 tags have the same class attached to them "fixed" let's see what that is.  Let's take a look at that

      You should have already learned a bit about different positioning types, relative, absolute,
      and fixed.  This class is relatively new to CSS and it automatically transitions from behaving
      like relative -- it sits within its parent element... until it would be scrolled off the page.
      then it behaves like fixed. 

      and while we're here, take a look at the next class, poem.  it has a margin of 80 pixels above it,
      that's a touch over an inch on 72 pixel per inch displays.

      we'll come back to the CSS, but take note that everything else here is a class, denoted by the "."
      before each selector.
      
      looking back at the html, the first line of the poem starts with a p tag,
      that p has two classes: letters and large.  then a span starts before the first word with
      class "vert". Every other word in the line has a span with the class vert.

      Back to the CSS:
      Large sets the font-size to 1.3 em.  ems are measued by the font size of the current element.  
      so 1 em is regular size of a p tag.  with 'large', it's 130% that size. a bit bigger.

      The letters class which is applied to the entire line has letter-spacing set,
      Setting the letter-spacing to .7 em means there's not a full letter's space between each letter. 
      Pretty close, though.

      The vert class adds two properties:
      writing-mode: veretical-lr which is for vertical text that is ordered left to right (some languages would go rl);
      and text-orientation upright means that the letters are rotated so that they are upright rather than rotated.
      
      I'll comment that line out and let's see how it changes.

      Let's look at the next class, turn.
      The turn class adds a transform property, rotating elements 7 degrees.  
      But the text seems to be turning more than 7, why? 
      Looking at the HTML,
      we see that each group of words has a div, but they aren't encapsulated like the last line, opening and closing
      around each word  Rather, they are nested, a div in a div in a div in a div,
      and since each of these divs has the class that adds rotation, they are adding together.
      I'm going to uncomment the border property to make it clearer how they 'stack'.

      This class also changes the display property of these spans from inline to inline-block.  Inline-block is similar to inline
      where content will flow without breaking a line, but it also obeys some positioning rules that inline elements don't.
      Try taking this rule out and see what happens.
      

      We have a similar dynamic with nested SPAN tags in the next line. the 'grow' class has a font-size of 2 em.
      double the font size. Since these are nested, the words increase by double each time another span is added.

      the next line makes use of the position property.
      an element with relative positioning will follow the normal "flow" of the document,
      the outer element here is positioned with relative, but the inner element is positioned with "absolute";
      this means it obeys top: left: right: bottom: for positioning. 

      There are spans around the first and last word, the huge class sets the font-size to 6 em.  If you were to nest
      these, the text would become so big you'd only see parts of a letter at a time. Something to experiment with!
      
      the next line has 3 classes attached to the p tag: huge and turn, which we've already looked at, and marker.
      Marker is the other half of the embed from google fonts. 

      this line has 'large' applied to the p tag, and then both vert and turn applied to the words "old gang" using a span.

      finally, we have a div with large applied to it, and a class we have't seen yet, 'perspective'
      perspective applied a transform, similar to the rotation I used in the turn class.
      The perspective property determines how close you are in the 3d space, and then rotating on the y axis 
      effectively tilts the contents toward us. the relationship between these numbers is a little noninuitive 
      and it took quite a bit of tweaking to not go off the page.
